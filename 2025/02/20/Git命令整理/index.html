Git 实用指南

一、Git 核心概念

  1. 三层架构：

    • 工作区（Working Directory）：本地可见文件
    • 暂存区（Staging Area/Index）：准备提交的变更
    • 版本库（Repository）：保存所有版本数据（包含本地仓库和远程仓库）

  2. 分布式特性：每个开发者都有完整的仓库副本

四、实用场景速查表

需求场景 推荐命令组合

恢复已删除文件 git checkout HEAD^ -- 文件名

查找引入bug的提交 git bisect + 测试脚本
同步远程已删除的分支 git fetch -p

五、最佳实践建议

  1. 提交规范

    • 使用约定式提交
    • 示例：feat(user): add password strength meter

  2. 分支管理策略

    • main 分支：稳定版本
    • dev 分支：日常开发
    • 特性分支：feature/xxx
    • 热修复分支：hotfix/xxx

  3. 协作流程优化

    • 每天开始工作前执行 git fetch
    • 推荐配置 pull.rebase = merges
    • 使用 git config --global rerere.enabled true 自动解决重复冲突

  4. 安全防护

    • 重要分支设置保护规则
    • 慎用 --force 参数
    • 定期执行 git gc 优化仓库

核心组件流程说明：

  1. 文件变更首先发生在工作区
  2. 使用 git add 将变更提交到暂存区
  3. 通过 git commit 永久保存到本地仓库
  4. git push/pull 实现与远程仓库的同步
  5. 分支操作（创建/切换）均在本地仓库完成

?? 深度提示：Git 内部使用四种对象类型（blob、tree、commit、tag）构建版本树，每个对象都有唯一的SHA-1哈希值标识。推荐使用 git cat-file -p [hash] 查看对象详细内容。

扩展建议：
使用 git log --all --graph --format=format:'%C(bold blue)%h%C(reset) - %C(white)%s%C(reset)' 可获得更清晰的版本树可视化效果。建议将其设置为 alias.visual 使用。

希望本指南能助您提升Git使用效率，建议结合官方文档（git-scm.com）深入探究高级功能。遇到合并冲突时，推荐使用 git mergetool 调用可视化工具解决。
