Golang面试题整理

基础语法

01 = 和 := 的区别？

  • =: 赋值操作符，仅用于已声明变量的赋值。
  • :=: 短变量声明，用于声明并初始化新变量，自动推断类型。仅限函数内部使用。

02 指针的作用

  • 存储变量内存地址，允许间接访问/修改变量值，减少数据拷贝成本，支持函数间共享数据。

03 Go 允许多个返回值吗？

  • 允许，常用于返回结果和错误，如 func f() (int, error)。

04 Go 有异常类型吗？

  • 无传统异常，使用 error 类型处理错误，panic 和 recover 处理运行时严重错误。

05 协程（Goroutine）

  • 轻量级线程，Go 运行时管理，启动成本低（几 KB）。用法：go func() {...}。

06 高效拼接字符串

  • 使用 strings.Builder 或 bytes.Buffer，避免频繁内存分配。

07 rune 类型

  • rune 是 int32 别名，表示一个 Unicode 码点。用于处理多字节字符（如中文）。

08 判断 map 是否包含 key

  • 使用 value, ok := m[key]，ok 为 true 时存在。

09 默认参数或可选参数

  • 不支持。替代方案：
    • 使用结构体参数：type Options struct { ... }
    • 可变参数：func f(args ...int)

10 defer 执行顺序

  • 按后进先出（LIFO）顺序执行。最后一个 defer 最先执行。

11 交换变量值

  • 直接交换：a, b = b, a。

13 判断切片相等

  • 循环比较元素，或使用 reflect.DeepEqual(s1, s2)。后者可能影响性能。

14 %v 与 %+v 区别

  • %v: 默认格式输出值。
  • %+v: 对结构体额外输出字段名。

  • 零内存占位符，用途：
    • 信号通道：ch := make(chan struct{})
    • 集合：set := make(map[string]struct{})

实现原理

01 init() 执行时机

  • 在包导入时自动执行，先于 main。多个包的 init 按依赖顺序执行。

02 局部变量分配位置

  • 由逃逸分析决定：若变量逃逸出函数则分配到堆，否则在栈。

03 两个 interface 可以比较吗？

  • 当动态类型是可比较类型（如 int、string）且值相等时，可比较，否则可能 panic。

04 两个 nil 可能不相等吗？

  • 是。接口值的 nil 需同时满足动态类型和值为 nil：

05 GC 工作原理

  • 三色标记法（并发标记-清扫）：
    1. 初始标记（STW）
    2. 并发标记（与程序并行）
    3. 重新标记（STW）
    4. 并发清除

06 返回局部变量指针安全吗？

  • 安全。编译器通过逃逸分析决定分配到堆，避免悬垂指针。

*07 T 调用 T 方法？

  • *T 可调用 T 的方法：编译器自动转换为 (&t).Method()。
  • *T 无法调用 T 的方法：除非 T 是可寻址的（如非临时变量）。

并发编程

01 无缓冲 vs 有缓冲 channel

  • 无缓冲：发送和接收需同步（阻塞直到对方准备好）。
  • 有缓冲：缓冲区未满时可异步发送，已满则阻塞。

02 协程泄露

  • 协程因阻塞（如未关闭的 channel）无法终止，导致资源无法释放。

03 限制线程数量

  • 使用 runtime.GOMAXPROCS(1)，限制并行线程数为 1，但通常不建议。

代码输出、作用域、defer

  • defer 参数：参数立即求值，但函数延迟执行。

  • 作用域覆盖：内层变量覆盖外层同名变量。
